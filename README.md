# ようかん TOP / アニメーション雛形

学習目的で実装した練習用プロジェクトです。
機能ごとにコードを分けてあるので、**アニメーション雛形（テンプレート）** としても流用できます。

> ⚠️ アニメーションの動き・ライブラリ構成は実在サイトの挙動を参考に学習したものですが、
> **画像・ロゴ・ブランド名（ようかん）・文言・商品名はすべてオリジナルのプレースホルダーに差し替え済み**です。
> 外部サイトの画像を直リンクしている箇所はありません（画像は `images/` 配下のローカル SVG）。
> 公開（GitHub Pages 等）してもそのまま使えますが、実案件では `images/` の画像を本番素材に差し替えてください。

## どんな動きが入っているか

ページを開くと、上から順に次の演出が流れます。

1. **イントロ** … 見出し「羊羹を、あたらしく。」が1文字ずつぼかしフェードで出る → 中央の羊羹がぼけて現れてピントが合う → ロゴが出る → カードが展開
2. **KV カルーセル** … 中央の商品カードが、カクッとした動き（`steps`）で自動で切り替わる無限ループ
3. **下部 info** … 商品名が1文字ずつ枠に打ち込まれ、「と」「の」が跳ね、「ようかん」がパラパラ跳ねる
4. **スクロール演出** … 下にスクロールすると「Three Features」の見出しが1文字ずつ立ち上がる
5. **ホバー演出（PC）** … 特徴の各項目にマウスを乗せると、画像カードがマウスに追従して現れる

## 使い方

ビルド不要。`index.html` をブラウザで開くだけで動きます。

```bash
# そのまま開く（macOS）
open index.html

# もしくは簡易サーバー経由（推奨：file:// の制限を避けられる）
python3 -m http.server
# → ブラウザで http://localhost:8000 を開く
```

**必要なもの**

- モダンブラウザ（Chrome / Safari / Edge / Firefox）
- インターネット接続（ライブラリと画像を外部から読み込むため）

## ファイル構成

```
.
├── index.html      … 構造（HTML）。ライブラリ・CSS・JS を読み込む
├── css/
│   └── style.css   … 全スタイル
├── js/
│   └── main.js     … 全スクリプト（モダン ES。5つの機能ブロックに分割）
└── README.md
```

**使用ライブラリ（本家と同じ／CDN 読み込み）**

| ライブラリ | 役割 |
|---|---|
| [GSAP](https://gsap.com/) + ScrollTrigger | イントロのタイムライン、スクロールでの文字出現 |
| [Lenis](https://lenis.darkroom.engineering/) | ページ全体のなめらかスクロール（慣性） |

> 本家では Swiper も使われていますが、それは別ページ（Collection の商品モーダル）用です。
> このトップページには不要なので入れていません。**KV カルーセルは本家・本再現とも自作**です。

## 仕組み（機能ブロック）

`js/main.js` は、独立した小さな塊（IIFE）が5つ並ぶだけの構成です。番号はファイル内のコメントと一致します。

| # | ブロック | やっていること |
|---|---|---|
| 0 | GSAP + Lenis 初期化 | ライブラリの起動と連携。イントロ中はスクロールを止める |
| 1 | KV カルーセル | カードを斜めに配置して自動送り。最後尾はクローンで前へ回し無限ループ |
| 2 | 下部 info | 商品名のタイプ表示、接続詞・サフィックスの跳ね、進行バー、数字ダイヤル |
| 3 | 見出しリビール | 文字を分解し、スクロールで下から押し上げて表示 |
| 4 | カーソル追従画像 | PC でホバー中、画像カードをマウスに追従させて表示 |
| 5 | 初回イントロ | 上記「どんな動き」の 1. を組み立てる |

### ブロックどうしのつなぎ方（カスタムイベント）

各ブロックは直接呼び合わず、**3つのイベント**でゆるくつながっています。組み替えるときはここだけ意識すればOKです。

```
[5 イントロ] ──(yokanKvSequenceStart)──▶ [1 カルーセル] … カードを展開
[1 カルーセル] ──(yokanKvSettled)──────▶ [2 info]       … 文言表示＋進行バー開始
[2 info] ──────(yokanKvAdvance)───────▶ [1 カルーセル] … 次のカードへ
```

## カスタマイズ早見表

「ここを変えればこうなる」をまとめました。

| 変えたいこと | 場所 | 目印 |
|---|---|---|
| 商品の画像・名前 | `index.html` | `<img src>` / `data-name-a` `data-name-b` `data-name-suffix` |
| ホバーで出る画像 | `index.html` | `data-hover-src` |
| 1枚あたりの表示時間 | `js/main.js`（2） | `KV_SEASON_MS`（既定 4000ms） |
| 文字を打つ速さ | `js/main.js`（2） | `KV_TYPE_CHAR_MS`（既定 92ms） |
| カードの間隔 | `js/main.js`（1） | `gap()` の `dx` / `dy` |
| ぼかしの強さ | `js/main.js`（1） | `look()` の `blur`（0 / 6 / 12px） |
| カクつきの段数 | `js/main.js`（1） | `STEP = 'steps(5, end)'` |
| スクロールの重さ | `js/main.js`（0） | `new Lenis({ lerp: 0.1 })`（小さいほど重い） |
| イントロの文字速度・間 | `index.html` | 見出しの `data-intro-*` 属性 |
| 色・配置・サイズ | `css/style.css` | クラス名（`.top-kv__*` など） |

## 転用するとき

他案件の雛形として使う手順です。

1. **中身を差し替える**
   画像（`<img src>` / `data-hover-src`）、見出し、カード名（`data-name-*`）を自分のものに。「再現版」ラベル（`.repro-badge`）は削除。
2. **クラス名を変える**（任意）
   `top-kv__` `top-features__` などのプレフィックスを案件名に一括置換すると、他CSSと衝突しにくくなります。
3. **要らない演出を外す**
   ブロックは独立しているので、不要な番号の塊ごと削除すればOK（例：ホバー演出だけ欲しい → ブロック4だけ残す）。
4. **ライブラリの取捨選択**
   - 慣性スクロール不要 → Lenis の `<script>` とブロック0の Lenis 部分を削除
   - GSAP が無くても、イントロ・リビールは **CSS だけで動くフォールバック** を実装済み

### そのまま使い回しやすい部品

- **文字リビール**（ブロック3 ＋ `char-mask` / `char-inner` の CSS）… どんな見出しにも付けられる
- **カーソル追従画像**（ブロック4 ＋ `.cursor-yokan` の CSS）… `data-hover-src` を付けた要素に効く
- **中央フォーカス型の無限カルーセル**（ブロック1）

---

## 注意点

- 画像・ライブラリを外部から読み込むため、**オフラインでは表示が崩れます**。実運用ではローカルに置いてください。
- KV 画像は白背景の JPG を `mix-blend-mode: multiply` で背景になじませています。背景色を変えると白フチが見えることがあります（透過 PNG 推奨）。
- `prefers-reduced-motion: reduce`（モーション軽減）設定の環境では、イントロを飛ばして即表示します。
